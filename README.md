## LAB TRabajando con la función print()

### La función print() es una de las directivas más fáciles de trabajar, lo que hace es imprimir en terminal lo que se le coloca en los argumentos. Esto pueden ser variables o cadenas de texto simples, pero en el caso de cadenas de texto simples siempre se deben colocar entre comillas debido a que esto le dice a python que no es código para ejecutar. Si no se utilizan estas python puede comenzar a buscar funciones o variables dentro de los argumentos, lo que da como resultado que no encuentre qué imprimir porque la cadena de texto normalmente no representa una variable del entorno. El error que resultó en mi caso cuando coloqué una cadena de texto sin sus respectivas comillas dice que si me olvidé de comas, lo que me dice que python pensó que mi cadena de texto eran variables que no podía entender por falta de comas. Por otro lado cuando lo que faltan son los paréntesis que le dicen a python que este es el argumento de la función print la ejecución se detiene, aborta por un error de sintaxis y te dice que si te has olvidado de los paréntesis de la función. Python admite tanto comillas simples como dobles para cadenas de texto simples con lestras, probablemente sea diferente si el argumento tiene una estructura más complicada, pero para cadenas de texto sirven ambas. En caso de que se pongan dos funciones print() en la misma línea python da otro error de sintaxis, al parecer para que algo así funcione tendría que tener algún tipo de estructura con una palabra reservada "in", pero aún no se ve ese tema.
